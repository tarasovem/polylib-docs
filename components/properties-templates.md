# Свойства и шаблоны

Свойства и шаблоны - центральная часть Polylib. Свойства описывают состояние, шаблон описывает UI на основе этого состояния.

## Объявление свойств

Свойства объявляются статически у компонента:

```js
static properties = {
  title: { type: String },
  count: { type: Number },
  disabled: { type: Boolean },
};
```

Практика:

- задавайте понятные имена свойств;
- используйте значения по умолчанию в `constructor`;
- храните UI-состояние в свойствах, а не в DOM.

## Шаблон компонента

Метод `render()` возвращает шаблон через `html`:

```js
render() {
  return html`
    <button ?disabled="[[disabled]]" on-click="onIncrement">+</button>
    <span>[[title]]: [[count]]</span>
  `;
}
```

## Биндинги

- `[[...]]` - one-way binding (данные из свойства в шаблон).
- `{{...}}` - two-way binding (двусторонний сценарий, используйте осознанно).

## Обновление состояния

Для изменения состояния используйте API компонента, а не прямые манипуляции DOM:

```js
onIncrement() {
  this.set("count", this.count + 1);
}
```

## Частые ошибки

- Изменение текста напрямую через `querySelector` вместо обновления свойства.
- Смешивание бизнес-логики и разметки в шаблоне.
- Отсутствие начальных значений в `constructor`.
